## 7. INTEGRATION PATTERNS

### HTMX + Go-Templ
- **Template rendering**: All pages use go-templ components
- **Dynamic content**: HTMX attributes (`hx-get`, `hx-post`, `hx-swap`, `hx-trigger`) trigger handlers
- **Event polling**: Agents/teams/tasks reload every 5-10s via `hx-trigger="every Ns"`
- **Form submission**: `hx-post` on project init, chat send, approvals

### SSE (Server-Sent Events)
- **Stream endpoint**: `GET /api/chat/stream` (line 108, server.go)
- **Handler**: `handleChatStream()` (line 372, handlers.go)
- **Client**: Opens EventSource in app.js, listens for events
- **Event types**: text, thinking, tool_start, tool_end, approval_needed, done, error, retry
- **Reconnection**: On disconnect, `handleChatReplay()` resends buffered events

### Authentication Flow
1. User visits `/login` → rendered by `handleLoginPage()`
2. Form submits to `POST /login` → `handleLogin()`
3. Password validated via `validatePassword()` (constant-time compare)
4. Token generated, stored in-memory map with 24h expiry
5. Cookie set: `claudio_token` (HttpOnly, SameSite=Lax)
6. Subsequent requests checked by `requireAuth()` middleware
7. Logout: `POST /logout` → clears token & cookie

### Session Lifecycle
1. **Create**: `POST /api/sessions/create` → new ProjectSession + DB persist
2. **Active**: User selects session, messages cached in memory
3. **Chat**: `POST /api/chat/send` → new query.Engine turn
4. **Stream**: `GET /api/chat/stream` → SSE events fed to browser
5. **Approval**: Tool approval blocks turn until user approves/denies
6. **History**: `GET /api/sessions/history` → reload chat on reconnect
7. **Delete**: `POST /api/sessions/delete` → remove from DB

## Key Architectural Notes

### No Traditional Frontend Build Step
- Templates are compiled at build time via go-templ compiler
- CSS is generated dynamically (Tailwind) or embedded (static.go)
- JS is embedded in static.go as a constant
- Static files served from embedded map or filesystem fallback

### Event-Driven Architecture
- SSE for real-time streaming (chat, tool execution)
- Buffered ring buffer (2048 events) for reconnection replay
- Event handler bridges query engine → browser

### Session Persistence
- Sessions stored in shared SQLite DB
- In-memory ProjectSession cache for fast lookups
- Multiple projects, multiple sessions per project

### Tool Approval Gate
- Deferred tools prompt user before execution
- Approval blocks query.Engine.Run() until decision
- Event emitted to browser, user responds via form POST

### Responsive Design
- Mobile sidebar (hidden on xs, visible on lg)
- Mobile menu toggle + drawer overlay
- Tailwind responsive classes + CSS media queries
